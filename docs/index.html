Bundled Data

The package vignette and examples use the bundled TCGA BRCA data.

library(multiRF)
data("tcga_brca")
data("tcga_brca_clinical")

names(tcga_brca)

Samples should already be aligned across blocks. The expected structure is one named list of omics matrices with samples in rows and features in columns.

Clustering Workflows

mrf3() is the compact entry point. Use it when you want the standard workflow without exposing every stage manually.

fit <- mrf3(
  tcga_brca,
  k = 4,
  ntree = 100,
  filter_mode = "none",
  run_imd = TRUE,
  seed = 529
)

Inspect the fitted object

summary(fit)
table(get_clusters(fit))
names(get_weights(fit))
get_top_vars(fit, n = 10)

Compare clustering backends

Similarity

similarity = default fused similarity view proximity = forest-proximity-driven branch enhanced_proximity = refined proximity branch

Variable Selection Methods

Variable selection is driven by IMD weights, so the normal pattern is to run IMD first and then enable the variable-selection stage.

fit_vs <- mrf3_fit(
  dat.list = tcga_brca,
  k = 4,
  ntree = 200,
  run_imd = TRUE,
  run_variable_selection = TRUE,
  variable_selection_args = list(
    method = "mixture",
    level = 0.05,
    normalized = TRUE
  ),
  seed = 529
)

Robust Clustering

Robust clustering means selecting variables first and then reclustering on the reduced representation.

fit_robust <- mrf3_fit(
  dat.list = tcga_brca,
  k = 4,
  ntree = 200,
  run_imd = TRUE,
  run_variable_selection = TRUE,
  run_robust_clustering = TRUE,
  variable_selection_args = list(method = "mixture"),
  seed = 529
)
table(fit_robust$robust_clusters)
summary(fit_robust)

Stability under resampling

stab <- mrf3_stability(
  fit_robust,
  branches = c("specific_shared", "robust_clustering"),
  n_rep = 20,
  sample_frac = 0.8,
  sample_mode = "subsample",
  run_consensus = TRUE,
  seed = 529
)

Method Comparison Cheatsheet

Clustering choices

  • similarity is the cleanest default when you want fused similarity matrices to drive clustering.
  • proximity is appropriate when you want to stay closer to random forest proximity information.
  • enhanced_proximity is the aggressive proximity-based option.
  • For branch-specific evaluation, inspect both fit$clusters and fit$robust_clusters.

Variable selection choices

  • filter is a practical baseline.
  • test is useful when you want significance-style selection.
  • mixture is useful when weights look like a signal/noise mixture.
  • thres is the most direct threshold-driven option.
cluster_metric_matrix(
